Technical Specification: "Elishelet" Web App (No-AI Runtime)1. OverviewProject Name: ElisheletRuntime Architecture: 100% Client-side (HTML5 Canvas / Vanilla JavaScript).Objective: Programmatically overlay user-inputted text on empty.png inside the boundaries of the white paper. The text must warp dynamically to match the paper's perspective, angle, and tilt, rendering instantly as the user types.2. Anchor Mapping & Geometry CalculationsTo warp flat text onto a skewed, 3D-angled piece of paper without AI, we must define a 2D Homography Transformation.Using the target image empty.png (dimensions: 1080 × 1920 pixels), we map the four corners of the white note to establish our canvas projection coordinate boundaries.2.1 Fixed Corner Coordinates (Target Plane on empty.png)The coordinates of the four corners of the paper in the native $1080 \times 1920$ resolution are:CornerCoordinates (X,Y) on empty.pngDescription$P_1$ (Top-Left)$(174, 1378)$Located near the left thumb crease$P_2$ (Top-Right)$(678, 1391)$Slightly lower due to the tilt of the page$P_3$ (Bottom-Right)$(676, 1756)$Bottom right corner above the right hand$P_4$ (Bottom-Left)$(183, 1742)$Bottom left corner above the left hand2.2 Inside Padding Box (Safe Rendering Zone)To prevent the text from running off the edges or being covered by the user's thumbs, we define an inset text rendering area ($5\%$ padding on all sides):Width of Target Note: $\approx 500 \text{ px}$Height of Target Note: $\approx 365 \text{ px}$Safe Zone Scale Factor: $0.90$ (centered relative to $P_1, P_2, P_3, P_4$).3. The Math: 4-Point Homography TransformationThe app will map a flat, virtual HTML5 canvas of dimensions $W \times H$ (e.g., $500 \times 365$) containing the rendered text onto the target quadrilateral $(P_1, P_2, P_3, P_4)$ using a $3 \times 3$ homography matrix $H$:$$\begin{bmatrix} x' \\ y' \\ 1 \end{bmatrix} \sim H \begin{bmatrix} x \\ y \\ 1 \end{bmatrix} = \begin{bmatrix} h_{00} & h_{01} & h_{02} \\ h_{10} & h_{11} & h_{12} \\ h_{20} & h_{21} & h_{22} \end{bmatrix} \begin{bmatrix} x \\ y \\ 1 \end{bmatrix}$$Implementation OptionsOption A (Pure CSS): Apply a transform: matrix3d(...) property to an HTML text overlay div positioned on top of the image container. This is extremely fast and lightweight.Option B (Canvas/Webgl API): Draw empty.png to a canvas, calculate the warp using a small JS library (e.g., OpenCV.js or a lightweight matrix utility), draw the text, and flatten the image for download.4. UI & Core Assets4.1 Asset Requirementsempty.png: The static background template ($1080 \times 1920$ px).Fonts (loaded via Google Fonts API):Assistant (Sans-Serif, default)Rubik (Rounded Sans-Serif)Heebo (Geometric)Frank Ruhl Libre (Serif)Alef (Dual-script Optimized)4.2 Control Controls (The Web Interface)Textarea: Multi-line input supporting standard Line Breaks.Text Direction: Segmented control button: [ LTR | RTL ].Note: Standard CSS direction: rtl or Canvas direction = "rtl" handles bidirectional text.Color Swatches:Deep Purple (#4D2B75 - mimicking the original marker)Charcoal Black (#1C1C1C)Royal Blue (#1D3557)Dark Red (#8B0000)Download Button: Instantly compiles the dynamic canvas to a .png file.5. Rendering Pipeline (Execution Flow)Every time the user types or alters a setting, the browser performs these steps in $< 16\text{ms}$ (real-time rendering):1. Draw background image "empty.png" onto the Master Canvas (1080x1920).
       |
       v
2. Render user text inside a virtual, offscreen 2D canvas (e.g., 500x365 px) 
   using chosen Font, Color, Alignment, and 5% safe-padding.
       |
       v
3. Apply 3D matrix transformation to warp the offscreen canvas 
   to the target coordinates (P1, P2, P3, P4).
       |
       v
4. Render the warped text layer on top of the Master Canvas 
   using the "Multiply" blend mode (mixes natural shadows and paper texture).
Why "Multiply" Blend Mode is CrucialInstead of drawing solid color pixels over the paper, using globalCompositeOperation = 'multiply' ensures that any subtle gradient shadows and paper grain present in empty.png show through the text. This mimics real marker ink without needing complex AI shaders.
